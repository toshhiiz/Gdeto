# ГДЕТО - Казахстанский маркетплейс недвижимости

Современное веб-приложение для поиска, размещения и управления объявлениями о недвижимости в Казахстане.

## 🎯 О проекте

ГДЕТО - это полнофункциональный маркетплейс недвижимости с минималистичным, современным дизайном. Приложение позволяет:

- 🏠 Быстро найти квартиру, дом или офис
- ➕ Разместить собственное объявление
- ❤️ Сохранить избранные объявления
- 📱 Удобно просматривать на любом устройстве
- 🔒 Безопасно управлять учетной записью

## 📋 Требования

### Frontend
- Node.js 18+
- npm или yarn
- React 18+
- Vite

### Backend
- Node.js 18+
- Express.js или аналогичный фреймворк
- MongoDB или PostgreSQL
- JWT для аутентификации

## 🚀 Быстрый старт

### Frontend

```bash
cd gdeto-frontend

# Установка зависимостей
npm install

# Запуск разработки
npm run dev

# Сборка для production
npm run build

# Preview production build
npm run preview
```

Фронтенд будет доступен на `http://localhost:5173`

### Backend

Подробная спецификация API находится в файле `BACKEND_API_SPEC.md`

```bash
# 1. Скопируйте BACKEND_API_SPEC.md как основу
# 2. Создайте новый проект:

mkdir gdeto-backend
cd gdeto-backend
npm init -y
npm install express mongoose jsonwebtoken bcryptjs cors dotenv multer

# 3. Следуйте спецификации в BACKEND_API_SPEC.md
# 4. Запустите сервер на порте 3000
```

Переменные окружения:
```env
PORT=3000
NODE_ENV=development
MONGODB_URI=mongodb://localhost:27017/gdeto
JWT_SECRET=your_secret_key
UPLOAD_DIR=./uploads
```

## 🎨 Дизайн

Приложение использует минималистичный, современный дизайн:

- **Цветовая схема**: Черный (#000) как основной цвет + светло-серый (#F0F0F0) для акцентов
- **Шрифт**: Inter - современный и чистый шрифт
- **Расчеты**: Простые и яркие, без лишних теней
- **Интерактивность**: Плавные переходы (0.2s ease)

## 👨‍💻 Разработка

### При разработке фронтенда

Все компоненты находятся в `src/components/`. При добавлении новых компонентов:

1. Создайте в соответствующей папке
2. Используйте существующие UI компоненты из `components/UI/`
3. Следуйте дизайн-системе (цвета, типография, расчеты)
4. Добавьте пропсы для гибкости

### При разработке бэкенда

1. Используйте спецификацию в `BACKEND_API_SPEC.md`
2. Следуйте структуре response (success, data, error)
3. Реализуйте все эндпоинты как описано
4. Добавьте обработку ошибок
5. Используйте CORS и переменные окружения
